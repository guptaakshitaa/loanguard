# LoanGuard AI — Production Credit Risk System

A full-stack ML system that scores loan applications in real time. Takes a borrower's financial profile, predicts default probability, converts it to a 300–850 credit score, and returns an **APPROVE / REVIEW / DECLINE** decision — with SHAP explanations and a full audit trail.

---

## Live Demo

Once running, open **`http://localhost:8000/docs`** for the interactive API.

## Quickstart

### Option A — Run locally

**1. Install dependencies**
```bash
pip install -r requirements.txt
```

**2. Train the model**
```bash
export PYTHONPATH=/path/to/loanguard
python3 ml/train.py
```
This generates 50k synthetic loan data, runs 50 Optuna trials, and saves `models/champion_latest.pkl`.

To use your own dataset:
```bash
python3 ml/train.py --data-path your_dataset.csv
```

**3. Start the API**
```bash
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

**4. Open the docs**
```
http://localhost:8000/docs
```

```bash
cd docker
docker-compose up --build
```

## ML Pipeline

```
Raw application
      ↓
Derived features (loan_to_income, rate × dti, ...)
      ↓
IV filtering — keeps features with IV ≥ 0.02
      ↓
WoE encoding — converts values to risk scores
      ↓
Imputation + scaling
      ↓
XGBoost (Optuna-tuned, 50 trials × 5-fold CV)
      ↓
Platt calibration — ensures probabilities are reliable
      ↓
PDO scorecard — probability → 300–850 score
      ↓
Decision: APPROVE (≥620) / REVIEW (540–619) / DECLINE (≤539)
```

## Explainability

Every `/explain` call returns three things:

**SHAP** — which features drove this specific decision and by how much
```json
"top_risk_factors": [
  {"feature": "rate_x_dti", "shap_impact": 0.312, "direction": "increases_risk", "rank": 1},
  {"feature": "debt_to_income", "shap_impact": 0.189, "direction": "increases_risk", "rank": 2}
]
```

**LIME** — local linear approximation of the model around this data point

**Counterfactual** — actionable recourse: what changes would flip the decision
```json
"counterfactual": [
  {
    "feature": "debt_to_income",
    "current_value": 0.75,
    "suggested_value": 0.56,
    "change_description": "Reduce your monthly debt-to-income ratio"
  }
]
```

## Monitoring

**Drift detection** (`GET /drift`) computes PSI for every feature against the training distribution:

| PSI | Status |
|-----|--------|
| < 0.10 | Stable |
| 0.10 – 0.20 | Monitor |
| ≥ 0.20 | Alert — consider retraining |

**Audit trail** — every prediction is logged to SQLite with full input features, score, decision, and latency for regulatory compliance (FCRA/ECOA).

## Configuration

All settings are overridable via environment variables:

| Variable | Default | Description |
|---|---|---|
| `LOANGUARD_OPTUNA_N_TRIALS` | 50 | Hyperparameter tuning trials |
| `LOANGUARD_MODEL_DIR` | `models` | Where to load/save model artifacts |
| `LOANGUARD_ENVIRONMENT` | `development` | `development` or `production` |
| `LOANGUARD_MLFLOW_TRACKING_URI` | `mlruns` | MLflow backend |
| `LOANGUARD_IV_FILTER_THRESHOLD` | `0.02` | Minimum IV to keep a feature |
| `LOANGUARD_THRESHOLD_APPROVE` | `0.30` | Max probability for APPROVE |
| `LOANGUARD_THRESHOLD_DECLINE` | `0.70` | Min probability for DECLINE |
